Consent

By using this app, and clicking the "Authenticate" button, you hereby consent to this Privacy Policy and agree to its terms.

Information that is collected

This web application requests information from the Google/YouTube Data API V3 to retrieve the unique generated ID string for each broadcast on your YouTube channel.

The only time that personal information is obtained is via Google's own login form/authentication prompt, which is a necessary to provide this tool access to what Google defines as "sensitive scope" information.

When you click the "Authenticate" button, you are taken to a prompt that asks you for your Google Account and Password in order to authorise access to your API data.

How we use your information

The information that is retrieved from your approval is used explictly for the following reasons.

  • Automatically fetch the ID string for your YouTube broadcasts
  • Add the fetched ID string to a URL to load into your browser to load that specific chat window of your YouTube livestream

Beyond that, this app doesn't collect or store anything else - because I don't give a shit about the other garbage on your account.

Log Files

The server that hosts the application follows a standard procedure of using log files. These logs are used for troubleshooting purposes when something goes wrong with loading the app, what specific part of the code that failed, and the timestamp in the server's local time. The rest, as I mentioned earlier, I don't give a shit about.

Cookies

In order to access the information from the YouTube Data API V3 resource, Google requires users to log in to their Google account, which creates a cookie on your local machine with some of that login data. The part that is used by this website specifically is a unique code, that is generated on authentication and expires when the window is closed, to then be traded for token to use the site.

If you're able to make sense of it, the entire process is available to be viewed publicly in Google's OAuth 2.0 documentation.
